Sessions Panel — Internal Design

How should the Sessions browser display and organize your history? Three approaches:

┌─ SESSIONS ─────────────────┐
│ ▼ Today                    │
│   ▶ my-project             │
│     ├─ main               │
│     ├─ refactor-branch    │
│     └─ experiment         │
│   ▶ auth-redesign          │
│     └─ main               │
│ ▼ Yesterday                │
│   ▶ api-planning           │
│   ▶ bug-hunt               │
│ ▶ Last Week               │
└────────────────────────────┘

A — Tree View (Session → Threads)

Sessions expand to show threads (branches). Date groups collapse. Mirrors the underlying session/fork data model. Most information-dense.

┌─ SESSIONS ─────────────────┐
│ ── Today ──────────────────│
│ ● my-project    14 msgs    │
│ ○ auth-redesign  7 msgs     │
│ ── Yesterday ──────────────│
│ ○ api-planning   22 msgs    │
│ ○ bug-hunt       5 msgs     │
│ ── Last Week ──────────────│
│ ○ refactor-plan  31 msgs    │
│ ○ design-review  9 msgs     │
└────────────────────────────┘

B — Flat List with Metadata

Sessions as a flat list grouped by date. Shows message count. Active session highlighted with filled dot. Clean, scannable. Threads accessed by entering the session.

┌─ SESSIONS ──── [S][E] ─────┐
│ ╔══════════════════════════│
│ ║ ● my-project             │
│ ║ main thread · 14 msgs    │
│ ║ 2 branches · ctx 82%     │
│ ╚══════════════════════════│
│ auth-redesign              │
│ main · 7 msgs · ctx 23%  │
│ ─────────────────────────  │
│ api-planning               │
│ main · 22 msgs · ctx 71% │
└────────────────────────────┘

C — Card View with Context %

Each session as a mini-card. Active session boxed and highlighted. Shows thread name, message count, branch count, and context %. Richest info but fewer sessions visible at once.

The panel also gets tabs: [S]essions and [E]xtensions (future). Click your preferred session list style.
