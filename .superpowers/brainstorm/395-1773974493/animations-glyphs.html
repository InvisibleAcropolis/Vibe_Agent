Animation & Glyph Style

The app already has hue-shifting borders and a braille spinner. What additional animation effects should be layered in for selections, transitions, and component separation?

A

Selection Flash — border lights up on focus

When you select a session, click a menu item, or focus the editor, the active component's border briefly brightens to full accent color then settles. Like a "confirm" pulse. Uses the existing borderActive color and breath animation.

╔══ inactive ══╗ → ╔══ ACTIVE ═══╗ → ╔══ focused ══╗
B

Block-fill Wipe — session/thread transitions

When switching sessions or threads, the chat area does a quick block-fill wipe using ░ ▒ ▓ █ characters before showing the new content. Classic terminal transition effect, ~3–5 frames at 80ms each.

frame 1: ░░░░░░░░░ → frame 2: ▒▒▒▒▒▒▒▒▒ → frame 3: ▓▓▓▓▓▓▓▓▓ → content
C

Separator Glyphs — visual dividers between zones

Use decorative glyph rows between major UI zones (header/chat, chat/editor, panel sections). Think ·─·─· or ╌╌╌╌╌ or ▸ ▸ ▸ pattern lines. Static or slowly animated (cycling direction).

╌╌╌╌╌╌╌╌╌╌╌╌╌╌╌╌╌╌╌╌╌╌╌╌╌╌╌╌╌╌╌╌╌╌╌╌╌╌╌╌╌╌╌╌╌╌╌╌╌╌
D

Streaming Hue Surge — border color intensifies while agent works

When the agent is streaming a response, the border hue-shift accelerates and saturates (already partially implemented). Extend this so the side panel border and header also pulse in sync — the whole chrome breathes together.

idle: ────── → thinking: ══════ → streaming: ══════
E

Typed Status Messages — status bar animates letter by letter

When a new status message appears (e.g. "Compacting context…", "Session loaded"), it types in character by character at ~40ms/char like a terminal output. Subtle but adds life to the status line.

S → Se → Ses → Sess → ... → Session loaded ✓
